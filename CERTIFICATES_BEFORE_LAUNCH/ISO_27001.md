# 🏢 ISO/IEC 27001

> The international standard for an Information Security Management System (ISMS). The global counterpart to SOC 2 — deferred until enterprise/global expansion.

| Field | Detail |
|-------|--------|
| **Overview** | Certifiable standard for establishing, operating, and improving an ISMS. |
| **Purpose** | Internationally recognized proof of systematic information security management. |
| **Category** | 🏢 Enterprise Only |
| **Why it is needed** | Preferred by EU/global + enterprise buyers; **not** needed for initial launch. |
| **Legally required?** | No. |
| **Technically required?** | No. |
| **When to implement** | Enterprise/global expansion phase. |
| **Priority** | 🏢 Deferred (enterprise) |
| **Estimated Cost** | $15k–$50k+ (consulting + certification body) + ongoing. |
| **Renewal** | 3-year cert cycle + annual surveillance audits. |
| **Official Website** | https://www.iso.org/standard/27001 |
| **Eligibility** | A functioning ISMS with risk assessments + controls. |

## Step-by-Step Process
1. Define ISMS scope + leadership commitment.
2. Risk assessment + treatment plan; implement Annex A controls.
3. Internal audit → certification body audit (Stage 1 + 2).
4. Maintain via surveillance audits.

## Required Documents
- ISMS scope, risk assessment/treatment, Statement of Applicability (SoA), policies, audit records.

## Implementation Guide
- Map this knowledge base + [../docs/SECURITY_PLAN.md](../docs/SECURITY_PLAN.md) to Annex A controls (significant overlap with SOC 2).

## Best Practices
- Pursue jointly with SOC 2 (shared evidence); use compliance tooling; right-size scope.

## Common Mistakes
- Over-scoping; documentation without operation; ignoring continual improvement.
