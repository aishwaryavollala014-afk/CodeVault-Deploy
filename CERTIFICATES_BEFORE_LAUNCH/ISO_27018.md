# 🏢 ISO/IEC 27018

> Cloud privacy controls for protecting PII in public clouds — an ISO 27001 extension. Deferred until enterprise/EU privacy assurance is needed.

| Field | Detail |
|-------|--------|
| **Overview** | Code of practice for protecting personally identifiable information (PII) in public-cloud environments. |
| **Purpose** | Assure customers that PII in the cloud is handled to a recognized standard. |
| **Category** | 🏢 Enterprise Only |
| **Why it is needed** | Privacy-sensitive enterprise/EU buyers may request it; **not** needed for launch. |
| **Legally required?** | No. |
| **Technically required?** | No. |
| **When to implement** | With ISO 27001/27701, for enterprise privacy assurance. |
| **Priority** | 🏢 Deferred (enterprise) |
| **Estimated Cost** | Incremental on ISO 27001. |
| **Renewal** | With the 27001 cycle. |
| **Official Website** | https://www.iso.org/standard/76559.html |
| **Eligibility** | ISO 27001 ISMS + cloud PII processing. |

## Step-by-Step Process
1. Establish ISO 27001 (+ ideally 27701).
2. Implement 27018 PII-in-cloud controls (consent, transparency, data location).
3. Certify as an extension.

## Required Documents
- ISO 27001/27701 docs + PII handling evidence.

## Implementation Guide
- Reuse [GDPR](GDPR.md) + [DATA_RETENTION_POLICY](DATA_RETENTION_POLICY.md) + [../docs/COMPLIANCE.md](../docs/COMPLIANCE.md).
